## Layer 4 Bug Review — 5 Bugs Found & Fixed

### Bug 1: [`run_layer4.py`](CHARTA/run_layer4.py:20) — `train({})` crashes with `KeyError`
**Problem:** `train(config)` expects `config["graphs_folder"]` but was called with empty dict `{}`.
**Fix:** Added `--graphs-folder` CLI argument and passed `{"graphs_folder": args.graphs_folder}` to `train()`.

### Bug 2: [`clinical_dataset.py`](CHARTA/src/layer4/clinical_dataset.py:18) — Improper `InMemoryDataset` inheritance
**Problem:** `ClinicalGraphDataset` extended `InMemoryDataset` but never called `super().__init__()` and didn't implement required `_download()` / `_process()` methods. PyG's `InMemoryDataset` expects a specific lifecycle that this class doesn't follow.
**Fix:** Changed to plain `class ClinicalGraphDataset:` — it's a simple list-based dataset with `__len__` / `__getitem__`, which works perfectly with PyG's `DataLoader` + `collate_fn`.

### Bug 3: [`trainer.py`](CHARTA/src/layer4/trainer.py:222) — `sklearn.train_test_split(dataset)` fails on HeteroData
**Problem:** `train_test_split(dataset, test_size=0.2)` tries `np.asarray()` on HeteroData objects, which crashes.
**Fix:** Split on integer indices instead: `indices = np.arange(len(dataset))`, then `train_test_split(indices, ...)`, and wrap with `torch.utils.data.Subset`.

### Bug 4: [`pipeline.py`](CHARTA/src/layer4/pipeline.py:102) — Device mismatch in `get_patient_embedding`
**Problem:** `get_patient_embedding(graph, model.encoder)` was called BEFORE `graph = graph.to(DEVICE)`. The model is on CUDA but the graph tensors are on CPU → crash.
**Fix:** Moved `graph = graph.to(DEVICE)` to line 102, BEFORE the embedding extraction call.

### Bug 5: [`trainer.py`](CHARTA/src/layer4/trainer.py:293) — `POSITIVE_CLASS_WEIGHT` defined but never used
**Problem:** `config.py` defines `POSITIVE_CLASS_WEIGHT = 3.0` for class imbalance, but `BCEWithLogitsLoss()` was created without it.
**Fix:** Added `pos_weight = torch.tensor([POSITIVE_CLASS_WEIGHT]).to(DEVICE)` and passed it to `BCEWithLogitsLoss(pos_weight=pos_weight)`.

### Step 4: Generate Labels CSV (Required Before Training)
The trainer expects `data/corpus_labels.csv` with columns: `patient_id, readmission, deterioration, medication`. If you don't have real labels yet, create synthetic ones:
```python
import csv, os
from pathlib import Path

# Collect all patient IDs from graph files
graph_dir = "/content/CHARTA/data/graphs"
patient_ids = []
for gf in sorted(Path(graph_dir).glob("*_graph.pt")):
    stem = gf.stem
    pid = stem.replace("_graph", "") if stem.endswith("_graph") else stem
    patient_ids.append(pid)

# Write labels CSV with random binary labels (replace with real labels later)
with open("/content/CHARTA/data/corpus_labels.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=["patient_id", "readmission", "deterioration", "medication"])
    writer.writeheader()
    for pid in patient_ids:
        writer.writerow({
            "patient_id": pid,
            "readmission": 1 if "cardiology" in pid else 0,
            "deterioration": 1 if "pulmonology" in pid else 0,
            "medication": 1 if "orthopedics" in pid else 0,
        })
print(f"Labels CSV created with {len(patient_ids)} patients")
```

### Step 5: Build FAISS Index
```bash
cd /content/CHARTA
python run_layer4.py --mode build_index --graphs data/graphs --output data/corpus_index
```
This loads all `*_graph.pt` files, extracts `patient_x` [1,768], applies random projection 768→256, and saves the FAISS index to `data/corpus_index/`.

### Step 6: Train GNN
```bash
python run_layer4.py --mode train --graphs-folder data/graphs
```
This runs 20 epochs of GraphSAGE + RAG + MultiTaskRiskModel training with:
- 80/10/10 train/val/test split
- BCEWithLogitsLoss (pos_weight=3.0)
- AdamW (lr=2e-4, weight_decay=0.01)
- CosineAnnealingLR scheduler
- Best model saved to `models/risk_heads/best_model.pt`

### ⚠️ Performance Note for T4
`format_rag_context()` runs ClinicalBERT inference for every sample in every batch. On a T4 with 16GB VRAM, this is slow (~5-10 min per epoch for small datasets). For faster training:
- Reduce `BATCH_SIZE` to 8 if you hit OOM
- Set `NUM_EPOCHS = 10` for initial experiments
- Pre-compute RAG contexts offline if dataset is large
